Installing gromacs-metatomic
============================

Installing `gromacs-metatomic` from the `metatensor` channel can be achieved by adding `metatensor` to your channels with:

```
conda config --add channels metatensor
conda config --set channel_priority strict
```

Once the `metatensor` channel has been enabled, `gromacs-metatomic` can be installed with `conda`:

```
conda install gromacs-metatomic
```

or with `mamba`:

```
mamba install gromacs-metatomic
```

It is possible to list all of the versions of `gromacs-metatomic` available on your platform with `conda`:

```
conda search gromacs-metatomic --channel metatensor
```

or with `mamba`:

```
mamba search gromacs-metatomic --channel metatensor
```

Alternatively, `mamba repoquery` may provide more information:

```
# Search all versions available on your platform:
mamba repoquery search gromacs-metatomic --channel metatensor

# List packages depending on `gromacs-metatomic`:
mamba repoquery whoneeds gromacs-metatomic --channel metatensor

# List dependencies of `gromacs-metatomic`:
mamba repoquery depends gromacs-metatomic --channel metatensor
```




Updating gromacs-metatomic-feedstock
====================================

If you would like to improve the gromacs-metatomic recipe or build a new
package version, please fork this repository and submit a PR. Upon submission,
your changes will be run on the appropriate platforms to give the reviewer an
opportunity to confirm that the changes result in a successful build. Once
merged, the recipe will be re-built and uploaded automatically to the
`metatensor` channel, whereupon the built conda packages will be available for
everybody to install and use from the `metatensor` channel.
Note that all branches in the conda-forge/gromacs-metatomic-feedstock are
immediately built and any created packages are uploaded, so PRs should be based
on branches in forks, and branches in the main repository should only be used to
build distinct package versions.

In order to produce a uniquely identifiable distribution:
 * If the version of a package **is not** being increased, please add or increase
   the [``build/number``](https://docs.conda.io/projects/conda-build/en/latest/resources/define-metadata.html#build-number-and-string).
 * If the version of a package **is** being increased, please remember to return
   the [``build/number``](https://docs.conda.io/projects/conda-build/en/latest/resources/define-metadata.html#build-number-and-string)
   back to 0.
